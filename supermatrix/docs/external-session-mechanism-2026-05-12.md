# SuperMatrix 外部群机制说明

版本日期：2026-05-12

本文说明 SuperMatrix 机器人在外部协作群中的响应规则、权限边界和使用方式。外部群的目标是支持公开、低风险的问题答复，同时避免泄露公司内部信息或误触发内部系统能力。

## 1. 什么是外部群

外部群指包含公司外部成员的飞书群。被标记为「外部」类型的 session（会话）会启用更严格的权限限制。

外部群机器人默认只做问题回答，不主动操作公司内部系统，不执行 SuperMatrix 的其它功能，不落地代码，不读取或暴露内部资料。

## 2. 响应规则

外部群机器人必须被 `@` 后才会响应。

| 场景 | 机器人行为 |
| --- | --- |
| 没有 `@` 机器人 | 不响应 |
| `@` 机器人并提出普通问题 | 按外部群权限回答 |
| 直接发送 `/help`、`/status` 等命令但没有 `@` | 不响应 |
| `@` 机器人并发送命令 | 进入权限判断 |

也就是说，在外部群中请使用类似：

```text
@SuperMatrix 你的问题
```

不要直接只发：

```text
/help
```

如果需要命令，也要先 `@` 机器人：

```text
@SuperMatrix /help
```

## 3. 身份判断

外部群里所有消息都会按发送人的平台身份做判断。系统只信任飞书提供的用户身份标识，不依赖昵称、备注名或消息文本里的自称。

身份分为两类：

| 身份 | 说明 |
| --- | --- |
| Owner（所有者） | 系统中预先配置的负责人身份 |
| External non-owner（外部非所有者） | 其它群成员 |

Owner 的具体身份标识由系统内部配置，不在外部群文档中公开展示。

## 4. Owner 消息规则

当消息来自系统配置的 Owner 时，机器人会把这条请求识别为 Owner 请求。

Owner 请求不受「外部成员不得访问公司内部信息」这条外部群限制阻断；机器人会按 Owner 的要求处理，但仍遵守系统级安全规则、平台权限和当前 session 的实际能力边界。

这意味着：

- Owner 可以在外部群中要求机器人解释、排查或处理与公司相关的问题。
- Owner 可以使用允许的命令。
- Owner 可以让机器人基于已授权上下文继续工作。
- Owner 的身份判断来自平台身份，不来自文本声明。

## 5. 非 Owner 消息规则

当消息来自非 Owner 成员时，机器人进入外部受限模式。

在外部受限模式下，机器人只回答当前问题，并且不得输出以下内容：

- 公司业务情况、经营数据、客户信息、供应商信息
- 公司人员信息、组织安排、内部决策
- 账号、密码、密钥、Token（访问令牌）等凭据
- SuperMatrix 的代码架构、内部实现、数据库结构、部署细节
- 内部 SOP（标准操作流程）、自动化流程、未公开工具能力
- 任何其它可能属于公司内部或敏感的信息

非 Owner 成员也不能通过外部群触发 SuperMatrix 的内部功能，例如：

- 不能执行内部 session 调度
- 不能读取历史附件或内部文件
- 不能让机器人修改代码、落地文件或操作公司系统
- 不能通过 slash command（斜杠命令）访问管理能力

## 6. 附件和历史信息

外部群对附件和历史信息有额外限制。

| 身份 | 附件处理 | 历史资料访问 |
| --- | --- | --- |
| Owner | 按授权能力处理 | 按授权能力处理 |
| 非 Owner | 不读取附件 | 不读取历史内部资料 |

非 Owner 发来的图片、文件或链接不会被当作内部工作资料处理。机器人只基于当前可公开问题做回答。

## 7. 命令使用

外部群中的命令同样必须先 `@` 机器人。

Owner 可以使用允许的命令，例如：

```text
@SuperMatrix /help
```

非 Owner 成员发送命令时，机器人不会开放内部管理能力。即使被 `@`，也只会按外部安全边界处理。

## 8. 设计目标

外部群机制的核心目标是：

- 防止机器人在外部群误回复未指向它的聊天内容
- 防止外部成员获取公司内部信息
- 防止外部成员触发内部系统操作
- 允许 Owner 在外部群里正常管理和使用机器人
- 保持外部协作场景下的可控、可审计、低风险

## 9. 使用建议

在外部群里使用机器人时，建议遵循以下方式：

- 需要机器人响应时，明确 `@SuperMatrix`
- 问题尽量描述清楚，不要只发单个关键词
- 外部成员只询问可公开、可对外回答的问题
- 涉及公司内部信息、账号权限、代码实现、系统操作的问题，由 Owner 发起

## 10. 当前适用范围

本文适用于已标记为「外部」类型的 SuperMatrix session。当前 xjbc 群已按外部群机制运行。
